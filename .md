### **ПОЛНАЯ АРХИТЕКТУРА ПРОЕКТА SNK (Sky New Kernel)**

---

#### **I. ФУНДАМЕНТАЛЬНЫЕ ПРИНЦИПЫ**
1. **Модульность на уровне компиляции** - сборка целевых систем только с необходимым функционалом
2. **Разделение времени сборки и выполнения** - различный менеджмент для разных этапов жизненного цикла
3. **Абстракция через интерфейсы** - единые стандарты для обеспечения переносимости

#### **II. ОРГАНИЗАЦИОННАЯ СТРУКТУРА**
- **Sky Collective** - головная организация
- **Основатели:**
  - *Тимофей Храмов (TimeStop34)* - архитектор ядра
  - *Ярослав Кострюков (Nimdacane)* - архитектор userland
- **Проектные псевдонимы** как элемент брендинга

#### **III. АРХИТЕКТУРНАЯ ИЕРАРХИЯ**

**Уровень 0: SNK Kernel (Абстрактное ядро-фреймворк)**
- Статус: эталонная реализация, набор интерфейсов
- Назначение: определение стандартов и API
- Особенность: невозможно загрузить напрямую

**Уровень 1: SNK [Ветка] (Ядра-дистрибутивы)**
- SNK Desktop → для работы с KurlukOS
- SNK IoT/Embedded → минимальная конфигурация  
- SNK Mobile → сбалансированная сборка
- Статус: загружаемые функционирующие ядра

**Уровень 2: Пользовательское пространство**
- KurlukOS - userland для SNK Desktop
- Разрабатывается отдельно под руководством Ярослава

#### **IV. СИСТЕМА МОДУЛЕЙ**

**Типология модулей:**

**A. По времени использования:**
1. **Compiling-Only модули** (управляются SNK BMM)
   - Исходные коды - для открытых компонентов
   - Бинарные объекты - для проприетарных драйверов

2. **Динамические модули** (управляются SNK MM Runtime)
   - Загружаемые на лету (.snm формат)
   - Аналог .ko в Linux

**B. По функциональности:**
1. **Аппаратные драйверы**
   - Ethernet/WiFi контроллеры
   - Драйверы шин (PCI/USB)
   - Графические ускорители

2. **Протокольные модули**  
   - TCP/IP стек
   - P2P стек (объединяет Bluetooth, WiFi Direct, радио)
   - Файловые системы

3. **Системные сервисы**
   - Менеджеры памяти
   - Планировщики (от кооперативных до вытесняющих)
   - Драйверы исполняемых файлов (UEFD)

#### **V. СИСТЕМА СБОРКИ И УПРАВЛЕНИЯ**

**SNK Build System:**
- **SNK Builder** - основной инструмент сборки
- **Make** - для компиляции отдельных компонентов
- Интеграция с системой управления модулями

**Двойная система менеджмента модулей:**
- **SNK BMM (Building Module Manager)** - для времени сборки
- **SNK MM Runtime** - для работающей системы

#### **VI. КЛЮЧЕВЫЕ ТЕХНОЛОГИИ**

**A. Совместимость:**
- **Linked API** - система трансляции системных вызовов
- Поддержка: Linux ABI, System V, FreeBSD, Mac OS
- **UEFD** - универсальный загрузчик исполняемых файлов

**B. Оптимизация:**
- Детальная настройка планировщиков под задачи
- Минимальные footprint для embedded-систем
- Специализированные конфигурации под архитектуры

#### **VII. ПРОЦЕСС РАЗРАБОТКИ И СОБОРКИ**

**Этап 1: Конфигурирование**
- Выбор базового профиля (Desktop/IoT/Mobile)
- Настройка через SNK Builder с привлечением BMM

**Этап 2: Разрешение зависимостей**  
- BMM анализирует и загружает compiling-only модули
- Проверка совместимости версий

**Этап 3: Компиляция**
- SNK Builder Orchestrates процесс сборки
- Использование Make для компонентов
- Сборка монолитного ядра с выбранными модулями

**Этап 4: Развертывание**
- Установка динамических модулей через SNK MM Runtime
- Обновление на лету без перезагрузки

#### **VIII. ПРЕИМУЩЕСТВА АРХИТЕКТУРЫ**

1. **Производительность** - отсутствие неиспользуемого кода
2. **Безопасность** - минимальная поверхность атаки  
3. **Гибкость** - адаптация под любые требования
4. **Совместимость** - мост к существующим экосистемам
5. **Масштабируемость** - от IoT до датацентров

#### **IX. СТАТУС РАЗРАБОТКИ**
- Активная разработка SNK Kernel и KurlukOS
- Формирование репозиториев модулей
- Разработка инструментов сборки и менеджмента
